## 1. 概述

### 1.1 模块定位

策略管理基础设施是所有策略模块的统一底座，为 F-007 网格交易策略管理（及后续 F-009~F-012）提供通用的策略生命周期管理、持仓管理、交易记录、策略仪表盘等基础能力。

### 1.2 与原始系统的对应

原始 codes-trading-strategy 插件中的以下能力需要在此模块中实现：

| 原始系统能力 | 对应本模块功能 | 说明 |
|-------------|---------------|------|
| 策略状态控制（DEBUG/PAUSE/CLEAR/CANCEL） | 策略状态管理 | 扩展为 5 种标准状态 |
| 持仓分析（holding_analysis） | 持仓管理 | 统一持仓计算，支持多策略类型 |
| 交易记录（trading_record） | 交易记录 | 统一交易记录，支持多种记录类型 |
| 总览视图（GTO） | 策略仪表盘 | 多策略聚合视图 |
| 现金余额（cash_balance） | 资金管理 | 用户级别的现金管理 |
| 标的总览（stock_overview） | 策略卡片 | 单策略摘要视图 |

### 1.3 设计原则

1. **统一入口**：所有策略类型（网格、定投、资产配置、红利）共用同一套 CRUD 接口
2. **类型隔离**：策略类型创建后不可更改，不同类型策略有独立的参数和视图
3. **状态驱动**：策略状态决定系统行为（监控、提醒、计算等）
4. **数据聚合**：支持跨策略的持仓汇总、收益汇总、资金汇总

### 2.2 功能详细说明

#### F008-1: 策略统一 CRUD

**用户故事**：
作为投资者，我想要统一管理我的所有策略（网格、定投、资产配置等），以便在一个地方查看和操作。

**验收标准**：
- [ ] 支持创建多种类型的策略（网格、定投、资产配置、红利）
- [ ] 策略类型创建后不可更改
- [ ] 支持查看策略列表（按类型/状态/创建时间筛选）
- [ ] 支持查看策略详情
- [ ] 支持修改策略名称、描述、参数
- [ ] 支持删除策略（保留历史记录）
- [ ] 支持为策略添加备注/标签

**业务规则**：
1. **策略类型**：
   - `grid`：网格交易策略（F-007）
   - `drip`：定投策略（F-010）
   - `allocation`：资产配置策略（F-010）
   - `dividend`：红利管理策略（F-009）
   - `valuation`：估值策略（F-011）
2. **创建约束**：
   - 同一标的可创建多个同类型策略
   - 策略名称唯一（用户级别）
   - 必须选择策略类型和标的
3. **删除约束**：
   - 删除后策略状态变为"已归档"，不再参与计算
   - 保留所有历史交易记录和持仓数据
   - 已归档策略可恢复

**策略数据结构**：

```json
{
  "strategy_id": "uuid",
  "user_id": "uuid",
  "name": "游戏ETF网格策略",
  "type": "grid",
  "symbol": "159869",
  "symbol_name": "游戏ETF",
  "status": "running",
  "params": {
    // 类型特定参数，由具体策略模块定义
    "target_price": 1.0,
    "max_slump_pct": 0.67,
    "sgrid_step_pct": 0.05,
    // ...
  },
  "created_at": "2024-01-01T00:00:00Z",
  "updated_at": "2024-01-01T00:00:00Z",
  "notes": "备注信息",
  "tags": ["高频", "ETF"]
}
```

#### F008-2: 策略状态管理

**用户故事**：
作为投资者，我想要控制策略的运行状态，以便在不同市场环境下灵活调整。

**验收标准**：
- [ ] 支持 5 种状态：草稿 → 运行中 → 暂停 → 已完成 → 已归档
- [ ] 状态流转有明确规则，不允许非法跳转
- [ ] 状态变更记录到操作日志
- [ ] 不同状态有不同的系统行为

**状态定义与行为**：

| 状态 | 代码 | 监控 | 提醒 | 计算 | 说明 |
|------|------|------|------|------|------|
| 草稿 | `draft` | 否 | 否 | 否 | 刚创建，参数未配置完成 |
| 运行中 | `running` | 是 | 是 | 是 | 正常监控和提醒 |
| 暂停 | `paused` | 部分 | 否 | 部分 | 暂停买入监控，继续卖出监控，计入回撤 |
| 已完成 | `completed` | 否 | 否 | 否 | 策略目标达成或手动结束，保留数据 |
| 已归档 | `archived` | 否 | 否 | 否 | 已删除或长期停用，仅保留历史记录 |

**状态流转图**：

```
                    ┌─────────┐
                    │  草稿    │
                    │ (draft) │
                    └────┬────┘
                         │ 配置完成
                         ▼
              ┌─────────────────────┐
              │      运行中          │
              │    (running)        │
              └──────┬──────┬───────┘
                     │      │
           手动暂停  │      │ 手动完成
                     ▼      ▼
              ┌─────────┐  ┌─────────┐
              │  暂停    │  │  已完成  │
              │(paused) │  │(completed)│
              └────┬────┘  └────┬────┘
                   │ 恢复运行    │
                   │            │ 删除
                   └──────┐    └──┐
                          ▼       ▼
                   ┌─────────────────┐
                   │     已归档       │
                   │   (archived)    │
                   └─────────────────┘
```

**业务规则**：
1. **草稿 → 运行中**：参数配置完成且通过校验
2. **运行中 → 暂停**：用户手动暂停，系统记录暂停时间
3. **暂停 → 运行中**：用户手动恢复，系统记录恢复时间
4. **运行中 → 已完成**：用户手动标记完成
5. **任何状态 → 已归档**：删除操作，保留数据但不再参与计算
6. **已完成/已归档 不可恢复为运行中**：需要创建新策略
7. **所有状态变更均为手动触发**，系统不自动切换策略状态

#### F008-3: 持仓管理

**用户故事**：
作为投资者，我想要查看我的持仓分布，以便了解风险暴露和资金占用。

**验收标准**：
- [ ] 显示用户级别的持仓总览（按标的聚合）
- [ ] 显示策略级别的持仓明细（按策略类型分类）
- [ ] 支持加权平均成本计算
- [ ] 支持同一标的多策略持仓合并显示
- [ ] 显示持仓盈亏、持仓占比、本金占比

**持仓数据结构**：

```json
{
  "user_id": "uuid",
  "symbol": "159869",
  "symbol_name": "游戏ETF",
  "total_hold": 50000,
  "total_cost": 45000,
  "current_price": 1.05,
  "current_value": 52500,
  "unrealized_pnl": 7500,
  "unrealized_pnl_pct": 0.1667,
  "strategies": [
    {
      "strategy_id": "uuid",
      "strategy_name": "游戏ETF网格策略",
      "strategy_type": "grid",
      "hold": 30000,
      "cost": 27000,
      "grid_types": {
        "小网": {"hold": 15000, "cost": 13500},
        "中网": {"hold": 10000, "cost": 9000},
        "大网": {"hold": 5000, "cost": 4500}
      }
    },
    {
      "strategy_id": "uuid2",
      "strategy_name": "游戏ETF定投",
      "strategy_type": "drip",
      "hold": 20000,
      "cost": 18000
    }
  ]
}
```

**业务规则**：
1. **成本计算**：加权平均法，总成本 / 总持仓
2. **盈亏计算**：当前市值 - 总成本
3. **策略内分类**：网格策略按小网/中网/大网/累积/补仓分类
4. **跨策略聚合**：同一标的的多策略持仓合并计算
5. **已归档策略**：持仓不计入总览，但保留历史记录

#### F008-4: 交易记录

**用户故事**：
作为投资者，我想要记录所有交易操作，以便追踪策略执行和计算收益。

**验收标准**：
- [ ] 支持多种交易类型：买入、卖出、红利、拆股、调整
- [ ] 记录交易时间、标的、策略、方向、价格、数量、金额、费用
- [ ] 支持关联到具体策略和网格层（如适用）
- [ ] 支持筛选（按标的/策略/时间/类型）
- [ ] 支持导出为 CSV/Excel
- [ ] 支持手动录入和批量导入

**交易记录数据结构**：

```json
{
  "trade_id": "uuid",
  "user_id": "uuid",
  "strategy_id": "uuid",
  "strategy_type": "grid",
  "symbol": "159869",
  "symbol_name": "游戏ETF",
  "trade_type": "buy",
  "grid_type": "小网",
  "grid_index": 0,
  "trade_date": "2024-01-23",
  "trade_price": 0.76,
  "trade_count": 12000,
  "trade_amount": 9120,
  "fee": 5,
  "paired_trade_id": null,
  "notes": "",
  "created_at": "2024-01-23T10:00:00Z"
}
```

**交易类型**：

| 类型 | 代码 | 说明 | 适用策略 |
|------|------|------|----------|
| 买入 | `buy` | 普通买入 | 所有 |
| 卖出 | `sell` | 普通卖出 | 所有 |
| 红利 | `dividend` | 现金分红 | 所有 |
| 拆股 | `split` | 股票拆分 | 所有 |
| 补仓 | `cover` | 额外补仓 | 网格 |
| 调整 | `adjust` | 参数调整涉及交易 | 所有 |
| 转入 | `transfer_in` | 资金/持仓转入 | 所有 |
| 转出 | `transfer_out` | 资金/持仓转出 | 所有 |

**业务规则**：
1. **不可删除**：交易记录一旦创建不可删除，可标记为"错误"并补充正确记录
2. **配对规则**：网格策略的 BUY 和 SELL 按同网格层自动配对
3. **费用记录**：交易费用单独记录，计入成本
4. **时间顺序**：交易记录按时间排序，影响持仓计算

#### F008-8: 资金管理

**用户故事**：
作为投资者，我想要管理我的资金分配，以便了解可用资金和已投入资金的分布。

**验收标准**：
- [ ] 显示用户现金余额
