## 1. 概述

### 1.1 模块定位

网格交易策略管理是 Strategy Compass 的核心策略模块，基于成熟的 Obsidian 插件版网格交易系统（codes-trading-strategy）进行 Web 化重构。保留原始系统的核心计算逻辑和三层网格架构，适配为在线多用户策略管理平台。

### 1.2 原始系统核心能力

原始 codes-trading-strategy 插件已实现：

| 能力 | 说明 |
|------|------|
| **三层网格架构** | 小网（5%步进）/ 中网（22%步进）/ 大网（52%步进） |
| **智能触发监控** | 自动识别已触发层、监控层、待触发层 |
| **实时价格更新** | 每秒从新浪财经获取实时行情 |
| **完整交易记录** | BUY/SELL/SHARE（红利/拆股）三种记录类型 |
| **持仓分析** | 按网格类型（小/中/大/累积/补仓）分类统计 |
| **收益分析** | 当前/回调/最大回撤三种情景的盈亏计算 |
| **回撤分析** | 20%-80% 多档回撤情景的压力测试 |
| **红利网格** | 自动计息、止盈价位计算、年化收益统计 |
| **调整记录** | 网格参数调整历史追踪 |
| **总览视图** | 多标的聚合监控、买入/卖出监控汇总 |

### 1.3 目标用户

**核心用户：网格交易者**
- 偏好震荡市做 T，持有 3-10 只网格标的
- 有正职工作，无法实时盯盘
- 核心诉求：三层网格自动监控、触发提醒、收益统计、交易纪律辅助

### 1.4 核心价值

1. **三层网格精细化管理** — 小网高频做 T + 中网波段 + 大网兜底
2. **自动触发监控** — 系统自动识别当前价格所在网格层，高亮显示监控层
3. **完整交易闭环** — 从网格创建 → 触发提醒 → 交易记录 → 收益分析
4. **多情景收益分析** — 当前/回调/最大回撤三种情景的盈亏预估
5. **纪律性辅助** — 通过提醒和记录，帮助用户遵守交易纪律

### 2.2 功能详细说明

#### F007-1: 网格策略创建

**用户故事**：
作为网格交易者，我想要创建一个新的网格交易策略，以便系统帮我跟踪和管理这个策略的执行。

**验收标准**：
- [ ] 用户可以选择标的（股票/ETF/基金）
- [ ] 用户设定首网价格（目标价格）
- [ ] 系统支持从自选股列表快速选择标的
- [ ] 创建后策略状态为"运行中"
- [ ] 自动获取标的当前价格作为参考

**业务规则**：
1. 一个用户可以为同一标的创建多个网格策略（不同参数）
2. 标的必须存在于系统支持的标的库中
3. 首网价格必须 > 0

#### F007-2: 网格参数配置

**用户故事**：
作为网格交易者，我想要配置三层网格的参数，以便系统按我的规则生成交易网格。

**验收标准**：
- [ ] 配置基础参数：首网金额、最大回撤、触发加点、交易精度、最小批量
- [ ] 配置小网参数：步进值（默认5%）、追加比例（默认5%）、保留利润数（默认4）
- [ ] 配置中网参数：步进值（默认22%）、追加比例（默认20%）、保留利润数（默认2）
- [ ] 配置大网参数：步进值（默认52%）、追加比例（默认50%）、保留利润数（默认1）
- [ ] 配置红利参数：计息年份、利率、步数、触发阈值
- [ ] 系统自动生成完整交易网格表
- [ ] 支持实时预览网格表

**业务规则**：
1. **三层网格定义**：
   - **小网**：步进 5%，高频做 T，保留 4 份利润
   - **中网**：步进 22%，波段操作，保留 2 份利润
   - **大网**：步进 52%，深度回调兜底，保留 1 份利润
2. **价格档位**：以首网价格为基准，按百分比递减计算各层买入价
3. **买入份数**：首网金额 × (1 + 层数 × 追加比例) ÷ 买入价，向下取整到最小批量
4. **卖出份数**：买入金额 - 保留利润 ÷ 卖出价，向下取整到最小批量
5. **触发加点**：买入触发价 = 买入价 + 加点；卖出触发价 = 卖出价 - 加点

#### F007-3: 交易网格展示

**用户故事**：
作为网格交易者，我想要查看完整的交易网格表，并清楚看到哪些层已触发、哪些层正在监控、哪些层待触发。

**验收标准**：
- [ ] 显示完整交易网格表（网格种类、价格档位、买入/卖出触发价、买入/卖出价格、份数、金额）
- [ ] **颜色标记**：
  - 黄色：买入监控层（当前价格接近，可能触发买入）
  - 灰色：买入已触发层（已执行买入）
  - 浅黄：卖出监控层（当前价格接近，可能触发卖出）
  - 深灰：卖出已触发层（已执行卖出）
- [ ] 显示当前标的价格和所在网格位置
- [ ] 显示距离下一触发层的价格差距

**业务规则**：
1. **买入监控层判定**：当前价格 × (1 - 最大涨幅) ≤ 买入价，且买入价 ≥ 最低买入价（首网价 × 最小买入百分比）
2. **卖出监控层判定**：当前价格 × (1 + 最大涨幅) ≥ 卖出价
3. **已触发层**：存在对应的 BUY/SELL 交易记录
4. **停止线**：低于最低买入价 5% 误差的网格层不再监控

#### F007-4: 触发提醒

**用户故事**：
作为网格交易者，我想要在价格触及网格线时收到提醒，以便及时执行交易。

**验收标准**：
- [ ] 价格触及买入/卖出触发价时，系统生成提醒
- [ ] 提醒包含：标的、网格种类、方向（买/卖）、触发价、目标价、建议份数
- [ ] 提醒状态：待处理/已确认/已忽略
- [ ] 支持站内消息提醒（MVP）
- [ ] 支持邮件/微信推送提醒（未来版本）

**业务规则**：
1. 提醒生成后，用户需要在系统中确认或忽略
2. 确认的提醒关联到交易记录
3. 同一层在短时间内重复触发只提醒一次（防抖动，建议 1 个交易日）
4. 未处理的提醒在策略卡片上显示 badge

#### F007-5: 交易记录

**用户故事**：
作为网格交易者，我想要记录每笔网格交易的实际成交信息，以便跟踪策略表现。

**验收标准**：
- [ ] 支持三种记录类型：BUY（买入）、SELL（卖出）、SHARE（红利/拆股）
- [ ] 记录交易时间、标的、网格种类、交易价格、交易份数
- [ ] 支持关联到对应的网格策略和触发提醒
- [ ] 支持手动录入和通过命令快速录入
- [ ] 显示每笔交易的盈亏（配对 BUY/SELL 后计算）

**业务规则**：
1. **BUY 记录**：买入方向，关联具体网格层
2. **SELL 记录**：卖出方向，必须与同层的 BUY 配对
3. **SHARE 记录**：
   - 红利：记录分红金额，减少持仓成本
   - 拆股：记录拆股比例，调整持仓数量
4. 交易记录一旦创建不可删除（可标记为"错误"并补充正确记录）
5. **配对逻辑**：同网格层的 BUY 和 SELL 自动配对，计算保留份数和持仓时间

**交易记录示例**：

```
BUY,2024-01-23,小网0,0.760,12000
SELL,2024-02-15,小网0,0.820,11500
→ 配对后：保留 500 份，持仓 23 天，占用本金 760×12000 - 820×11500 = 920000 - 943000 = -23000（盈利）
```

#### F007-6: 持仓分析

**用户故事**：
作为网格交易者，我想要了解我的持仓按网格类型是如何分布的，以便评估风险暴露。

**验收标准**：
- [ ] 按网格类型分类统计：小网/中网/大网/累积/补仓
- [ ] 显示每类的：持仓股数、占用本金、持仓金额、持仓均价、当前价格、持仓盈亏
- [ ] 显示清格盈利（按最高卖出价计算）和清仓盈利（按平均清仓价计算）
- [ ] 显示持仓占比和本金占比
- [ ] 汇总全部持仓的总览数据

**业务规则**：
1. **小网/中网/大网**：按交易记录中的网格种类分类统计
2. **累积**：未配对的 BUY 记录（长期持有部分）
3. **补仓**：以"补仓"开头的特殊 BUY 记录
4. 成本价采用加权平均法计算
5. 清格盈利 = 持仓数量 × 最高卖出价 - 占用本金
6. 清仓盈利 = 持仓数量 × 平均清仓价 - 占用本金

#### F007-7: 收益分析

**用户故事**：
作为网格交易者，我想要查看在不同市场情景下的收益预估，以便了解策略的稳健性。

**验收标准**：
- [ ] 显示三种情景的持仓分析：
  - **当前持仓**：基于当前价格
  - **回调持仓**：基于回调价格（首网价 × 回调百分比）
  - **最大持仓**：基于最大回撤价格（首网价 × 最小买入百分比）
- [ ] 每种情景显示：持仓股数、占用本金、持仓金额、持仓均价、实际价格、持仓盈亏
- [ ] 每种情景显示清格、止盈、清仓三种卖出策略的预估收益
- [ ] 显示投入资金、账面资金、投入盈亏、本金仓位

**业务规则**：
1. **当前持仓**：基于当前市场价格计算
2. **回调持仓**：假设价格回调到 BOTTOM_BUY_PCT（默认 20% 回撤）时的持仓
3. **最大持仓**：假设价格跌到 MINIMUM_BUY_PCT（默认 10% 回撤）时的最大持仓
4. **清格**：按最高卖出价（empty_price）卖出所有持仓
5. **止盈**：按第一档清仓价（take_profit_price）卖出 40% 持仓
6. **清仓**：按平均清仓价（clear_avg_price）卖出全部持仓

#### F007-8: 回撤分析

**用户故事**：
作为网格交易者，我想要了解在不同回撤幅度下的策略表现，以便评估风险承受能力。

**验收标准**：
- [ ] 支持 20%/30%/40%/50%/60%/70%/80% 多档回撤分析
- [ ] 每档显示：回撤比例、投入资金、持仓份额、持仓金额、持仓成本
- [ ] 每档显示：亏损金额、亏损比例、所需涨幅回本
- [ ] 每档显示反弹盈利：持仓份额、持仓金额、占用本金、持仓成本、浮盈比例、网格盈利、总浮盈比

**业务规则**：
1. 假设价格从首网价回撤指定比例，计算该价格以下所有网格层的买入总和
2. 反弹盈利按首网价 × (1 + 小网步进) 计算卖出价
3. 网格盈利 = 卖出金额 - 买入金额

#### F007-9: 红利网格

**用户故事**：
作为网格交易者，我想要跟踪红利再投资策略，以便实现复利增长。

**验收标准**：
- [ ] 显示历史红利记录：计息日期、当时持仓、当时价位、买入价格、买入份数、买入金额
- [ ] 显示投入比例、止盈价位、止盈价格、卖出价格、卖出份数、卖出金额、卖出收益、年化收益
- [ ] 自动计算下一计息日的预估数据
- [ ] 颜色标记：监控中（黄色）、已触发（灰色）、已卖出（浅黄）

**业务规则**：
1. **计息触发**：当前价格/首网价格 < INTEREST_TRIGGER（默认 0.85）时启动计息
2. **买入份数计算**：(总持仓 × 利率) ÷ (止盈比例/当前比例 - 1) ÷ 当前价格
3. **止盈比例**：当前比例的下一个 0.1 档位（如当前 0.951 → 止盈 1.1）
4. **年化收益**：(卖出收益 - 买入成本) / 当时持仓

**业务规则**：

3. 不涉及交易的调整显示 "-"

#### F007-11: 价格分析

**用户故事**：
作为网格交易者，我想要了解标的的历史价格走势，以便评估当前价格位置。

**验收标准**：
- [ ] 显示 1 年期价格分析：开始日期、结束日期、最低价、最高价、现价、现价回撤、现价涨幅
- [ ] 显示 5 年期价格分析
- [ ] 显示全生命周期价格分析
- [ ] 显示首网价相对于历史价格的回撤和涨幅

**业务规则**：
1. 数据来源：标的上市至今的历史价格
2. 回撤 = (现价 - 最低价) / 最低价
3. 涨幅 = (现价 - 最高价) / 最高价

#### F007-12: 网格策略编辑/暂停/删除

**用户故事**：
作为网格交易者，我想要灵活管理已有的网格策略。

**验收标准**：
- [ ] 支持修改策略参数（首网价格、网格参数等）
- [ ] 支持暂停/恢复策略监控
- [ ] 支持标记为已清盘（不再监控，保留数据）
- [ ] 支持标记为预备取消（仅显示总览，不监控）
- [ ] 支持删除策略（保留交易记录）

**业务规则**：
1. **运行中**：正常监控，生成提醒
2. **暂停**：保留所有数据，暂停买入监控，继续卖出监控
3. **已清盘**：不再监控，不计入回撤数据，保留完整记录用于复盘
4. **预备取消**：仅显示在总览中，不参与监控和计算
5. **调试模式**：is_debug=true，数据全部忽略，用于测试

#### F007-13: 批量导入/导出

**用户故事**：
作为从 Obsidian 插件迁移的用户，我想要批量导入现有的网格策略数据。

**验收标准**：
- [ ] 支持导入 codes-trading-strategy 的 .gtv 文件格式
- [ ] 支持导入交易记录（BUY/SELL/SHARE）
- [ ] 支持导入调整记录
- [ ] 支持导出策略数据为标准格式

## 3. 非功能需求

### 3.1 性能要求

- 网格状态检查：每 5 分钟扫描一次（非实时，原始系统为每秒）
- 提醒生成延迟：触及网格线后 5 分钟内生成提醒
- 页面加载时间：策略列表 < 1s，策略详情 < 2s
- 交易网格表渲染：< 500ms（100 行以内）

### 3.2 可用性要求

- 交易网格表颜色标记清晰，一眼识别监控状态
- 策略卡片显示关键信息：标的、当前层、盈亏、待处理提醒数
- 提醒操作简洁（一键确认/忽略）
- 支持命令行快速录入交易记录（如原始系统的 `add_record BUY,2024-01-23,小网0,0.760,12000`）

## 6. 技术约束

- 网格状态检查使用定时任务（celery/cron），非实时推送（原始系统为每秒轮询）
- 价格数据来自 Module 1 的统一数据接口

## 11. 附录

### 11.1 术语表

| 术语 | 定义 |
|------|------|
| 首网价格 | 网格策略的基准价格，所有网格层以此计算 |
| 小网 | 步进 5% 的细粒度网格，高频做 T |
| 中网 | 步进 22% 的中粒度网格，波段操作 |
| 大网 | 步进 52% 的粗粒度网格，深度回调兜底 |
| 价格档位 | 相对于首网价格的百分比（如 95% = 首网价 × 0.95） |
| 触发加点 | 买入触发价 = 买入价 + 加点；卖出触发价 = 卖出价 - 加点 |
| 保留利润 | 每网卖出时保留的份数，用于长期持有 |
| 清格价 | 最高卖出价，用于计算清格盈利 |
| 清仓价 | 平均清仓价格，用于计算清仓盈利 |
| 计息 | 红利再投资策略，价格下跌时自动买入 |
