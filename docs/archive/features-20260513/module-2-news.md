## 1. 概述

### 1.1 模块定位

新闻聚合模块是Strategy Compass的信息层，负责获取、处理、存储和展示金融新闻。为投资者提供及时、重要的市场信息，辅助投资决策。

### 1.2 目标用户

- 需要及时了解市场动态的投资者
- 需要重要新闻提醒的用户
- 需要新闻与持仓关联分析的用户

### 1.3 核心价值

- 提供及时、准确的金融新闻
- 筛选重要新闻，减少信息噪音
- 支持新闻与持仓关联分析
- 提供新闻历史查询和统计

### 2.2 功能详细说明

#### F001: 新闻数据获取

**用户故事**：
作为投资者，我想要获取及时的金融新闻，以便了解市场动态。

**验收标准**：
- [ ] 支持7×24小时新闻获取
- [ ] 新闻延迟不超过5分钟
- [ ] 支持增量更新（只获取新新闻）
- [ ] 获取失败时自动重试

**业务规则**：
1. 使用金十数据flash_newest.js接口（无鉴权，50条）
2. 每5分钟同步一次新闻数据
3. 支持新闻去重（基于新闻ID）
4. 获取失败时记录日志并告警

#### F002: 新闻数据清洗

**用户故事**：
作为投资者，我想要看到干净、易读的新闻内容，以便快速获取信息。

**验收标准**：
- [ ] 清除HTML标签和特殊字符
- [ ] 统一新闻格式
- [ ] 提取关键信息（时间、重要性）
- [ ] 支持新闻分类

**业务规则**：
1. 清除所有HTML标签，保留纯文本
2. 重要性映射：1-5级，5级最重要
3. 时间格式统一为ISO 8601
4. 支持新闻分类标签

#### F003: 新闻存储管理

**用户故事**：
作为系统，我需要存储和管理新闻数据，以便提供历史查询。

**验收标准**：
- [ ] 支持新闻存储和更新
- [ ] 支持新闻清理（72小时TTL）
- [ ] 支持新闻备份和恢复
- [ ] 存储空间合理使用

**业务规则**：
1. 新闻数据保留72小时，过期自动清理
2. 支持新闻数据备份
3. 存储空间监控和告警
4. 支持新闻数据导出

#### F004: 新闻列表展示

**用户故事**：
作为投资者，我想要查看新闻列表，以便了解最新动态。

**验收标准**：
- [ ] 支持分页加载
- [ ] 支持重要性筛选
- [ ] 支持时间排序
- [ ] 支持新闻详情查看

**业务规则**：
1. 默认按时间倒序排列
2. 支持按重要性筛选（1-5级）
3. 分页加载，每页20条
4. 支持下拉刷新

#### F005: 重要新闻提醒

**用户故事**：
作为投资者，我想要及时收到重要新闻提醒，以便快速响应。

**验收标准**：
- [ ] 重要新闻实时推送
- [ ] 支持提醒设置（重要性级别）
- [ ] 支持提醒方式（声音、震动）
- [ ] 支持提醒历史查看

**业务规则**：
1. 重要性4级以上的新闻自动提醒
2. 支持自定义提醒级别
3. 提醒方式可配置
4. 提醒历史保留24小时

#### F006: 新闻统计分析

**用户故事**：
作为投资者，我想要了解新闻统计信息，以便分析市场情绪。

**验收标准**：
- [ ] 新闻数量统计（按小时/天）
- [ ] 重要性分布统计
- [ ] 新闻趋势分析
- [ ] 统计图表展示

**业务规则**：
1. 统计最近24小时/7天/30天的新闻数量
2. 分析重要性分布
3. 识别新闻趋势（增加/减少）
4. 提供可视化图表

#### F007: 新闻关联分析

**用户故事**：
作为投资者，我想要了解新闻与持仓的关联，以便评估影响。

**验收标准**：
- [ ] 新闻与股票关联匹配
- [ ] 影响程度评估
- [ ] 关联新闻优先展示
- [ ] 历史关联分析

**业务规则**：
1. 基于股票名称匹配新闻内容
2. 评估新闻对股票的影响程度
3. 关联新闻优先展示
4. 提供历史关联分析报告

## 3. 非功能需求

### 3.1 性能要求

- 新闻获取延迟：< 5分钟
- 新闻列表加载：< 1秒
- 新闻搜索响应：< 500ms
- 并发用户支持：100+

### 3.2 安全要求

- 数据来源安全：验证数据源可靠性
- 内容安全：过滤有害信息
- 访问控制：基于用户权限的访问控制
- 数据加密：敏感信息加密存储

### 3.3 可用性要求

- 界面友好：简洁清晰的新闻展示
- 操作便捷：快速浏览和筛选
- 信息准确：新闻内容准确无误
- 更新及时：新闻更新及时

### 3.4 兼容性要求

- 设备兼容：桌面端和移动端适配
- 浏览器兼容：主流浏览器支持
- 网络兼容：弱网环境下可用

## 4. 数据需求

### 4.1 数据实体

| 实体名称 | 描述 | 关键属性 |
|----------|------|----------|
| News | 新闻基本信息 | id, title, content, source, importance, published_at |
| NewsTag | 新闻标签 | news_id, tag_name, tag_type |
| NewsStat | 新闻统计 | date, hour, count, importance_dist |
| NewsAlert | 新闻提醒 | user_id, news_id, alert_type, created_at |

### 4.2 数据关系

- News 1:N NewsTag（新闻与标签一对多）
- News 1:N NewsStat（新闻与统计一对多）
- User 1:N NewsAlert（用户与提醒一对多）

### 4.3 数据流

1. 金十数据 → 新闻获取服务 → 数据清洗 → 数据存储
2. 用户请求 → 新闻查询服务 → 数据返回 → 前端展示
3. 重要新闻 → 提醒服务 → 用户推送 → 提醒记录
