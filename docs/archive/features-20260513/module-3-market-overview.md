## 1. 概述

### 1.1 模块定位

市场概览模块是Strategy Compass的展示层，负责整合数据服务和新闻聚合的数据，为投资者提供直观、全面的市场概览。作为应用层核心模块，为用户提供一站式的市场信息查看体验。

### 1.2 目标用户

- 需要快速了解市场全貌的投资者
- 需要查看核心指数表现的用户
- 需要查看重要新闻的用户
- 需要查看估值分布的用户

### 1.3 核心价值

- 提供一站式的市场信息查看
- 直观展示市场整体状况
- 支持个性化关注和配置
- 辅助投资决策

### 2.2 功能详细说明

#### F001: 概览页面

**用户故事**：
作为投资者，我想要快速了解市场全貌，以便做出投资决策。

**验收标准**：
- [ ] 展示估值区间分布统计
- [ ] 展示核心指数卡片网格
- [ ] 展示重要快讯
- [ ] 支持快速导航到详细页面

**业务规则**：
1. 默认显示概览页面
2. 估值分布按区间统计（低估/适中/高估）
3. 核心指数卡片显示价格、涨跌幅、估值
4. 重要快讯显示最新5条

#### F002: 大盘页面

**用户故事**：
作为投资者，我想要查看大盘指数详细信息，以便分析市场走势。

**验收标准**：
- [ ] 支持A股/港股/美股分组
- [ ] 显示详细指数卡片
- [ ] 支持指数详情查看
- [ ] 支持实时行情更新

**业务规则**：
1. 按市场分组：A股、港股、美股
2. 每个指数卡片显示：价格、涨跌幅、估值、回撤
3. 点击卡片查看详情弹窗
4. 实时行情数据分钟级更新

#### F003: 板块页面

**用户故事**：
作为投资者，我想要查看板块数据，以便分析行业轮动。

**验收标准**：
- [ ] 展示行业板块数据
- [ ] 展示概念板块数据
- [ ] 支持板块排序
- [ ] 支持板块详情查看

**业务规则**：
1. 按行业分类和概念分类组织
2. 显示板块涨跌幅、成交额
3. 支持按涨跌幅、成交额排序
4. 点击板块查看详情

#### F004: 指数卡片

**用户故事**：
作为投资者，我想要快速查看指数关键信息，以便了解市场状况。

**验收标准**：
- [ ] 显示指数名称和代码
- [ ] 显示当前价格和涨跌幅
- [ ] 显示估值信息
- [ ] 显示回撤信息

**业务规则**：
1. 涨跌幅颜色标识（红涨绿跌）
2. 估值信息显示PE/PB百分位
3. 回撤信息显示当前回撤和最大回撤
4. 支持点击查看详情

#### F005: 指数详情弹窗

**用户故事**：
作为投资者，我想要查看指数详细信息，以便深入分析。

**验收标准**：
- [ ] 显示K线图表
- [ ] 显示估值指标
- [ ] 显示风险指标
- [ ] 支持时间范围选择

**业务规则**：
1. K线图表支持日/周/月切换
2. 估值指标显示PE/PB历史百分位
3. 风险指标显示回撤、波动率、夏普比率
4. 支持1个月/3个月/1年/全部时间范围

#### F006: 新闻快讯展示

**用户故事**：
作为投资者，我想要查看重要新闻快讯，以便了解市场动态。

**验收标准**：
- [ ] 显示最新重要新闻
- [ ] 显示新闻时间
- [ ] 显示新闻重要性
- [ ] 支持新闻详情查看

**业务规则**：
1. 显示最新5条重要新闻
2. 新闻按时间倒序排列
3. 重要性标识（1-5级）
4. 点击新闻查看详情

#### F007: 估值分布统计

**用户故事**：
作为投资者，我想要了解市场估值分布，以便判断市场整体估值水平。

**验收标准**：
- [ ] 显示估值区间分布
- [ ] 显示低估/适中/高估比例
- [ ] 显示估值趋势
- [ ] 支持历史对比

**业务规则**：
1. 估值区间：低估（≤30%）、适中（30%-70%）、高估（≥70%）
2. 显示各区间指数数量
3. 显示估值趋势变化
4. 支持与历史估值对比

## 3. 非功能需求

### 3.1 性能要求

- 页面加载时间：< 2秒
- 数据刷新延迟：< 1分钟
- 交互响应时间：< 500ms
- 并发用户支持：100+

### 3.2 安全要求

- 数据访问控制：基于用户权限
- 数据传输加密：HTTPS协议
- 敏感信息保护：用户持仓信息加密
- 操作审计：关键操作记录日志

### 3.3 可用性要求

- 界面友好：简洁直观的界面设计
- 操作便捷：3步内完成常用操作
- 信息清晰：关键信息突出显示
- 响应迅速：快速响应和刷新

### 3.4 兼容性要求

- 设备兼容：桌面端优先，移动端适配
- 浏览器兼容：Chrome 90+, Firefox 88+, Safari 14+
- 分辨率兼容：支持多种屏幕分辨率

## 4. 数据需求

### 4.1 数据实体

| 实体名称 | 描述 | 关键属性 |
|----------|------|----------|
| MarketOverview | 市场概览数据 | date, index_count, valuation_dist, news_count |
| IndexCard | 指数卡片数据 | code, name, price, change, change_pct, pe, pb |
| BoardData | 板块数据 | code, name, change, change_pct, volume |
| NewsFlash | 新闻快讯 | id, title, importance, published_at |

### 4.2 数据关系

- MarketOverview 1:N IndexCard（概览与指数卡片一对多）
- MarketOverview 1:N BoardData（概览与板块数据一对多）
- MarketOverview 1:N NewsFlash（概览与新闻快讯一对多）

### 4.3 数据流

1. 数据服务 → 市场概览服务 → 前端展示
2. 新闻服务 → 市场概览服务 → 前端展示
3. 用户操作 → 前端交互 → 数据刷新
