## 1. 概述

### 1.1 模块定位

用户系统是Strategy Compass的基础模块，负责用户认证、授权、个人配置和关注管理。作为横跨数据层的用户隔离层，为其他模块提供用户上下文和个性化数据存储。

### 1.2 目标用户

- 个人投资者
- 需要管理自选股和关注板块的用户
- 需要个性化投资策略配置的用户

### 1.3 核心价值

- 提供安全的用户认证和授权机制
- 支持个性化配置和关注管理
- 为其他模块提供用户上下文隔离

### 2.2 功能详细说明

#### F001: 微信OAuth登录

**用户故事**：
作为投资者，我想要通过微信账号快速登录，以便无需记忆额外的账号密码。

**验收标准**：
- [ ] 支持微信OAuth2.0授权流程
- [ ] 首次登录自动创建用户账号
- [ ] 已有账号直接登录
- [ ] 登录状态保持7天

**业务规则**：
1. 微信OAuth回调地址必须配置正确
2. 用户微信ID作为唯一标识
3. 登录失败时提供明确错误提示

#### F002: 邀请码注册

**用户故事**：
作为新用户，我想要通过邀请码完成注册，以便获得系统访问权限。

**验收标准**：
- [ ] 支持6位数字邀请码
- [ ] 邀请码一次性使用
- [ ] 注册成功后自动登录
- [ ] 邀请码无效时提示错误

**业务规则**：
1. 邀请码由管理员生成
2. 每个邀请码只能使用一次
3. 邀请码有效期为30天

#### F003: 用户信息管理

**用户故事**：
作为用户，我想要查看和修改个人信息，以便保持信息的准确性。

**验收标准**：
- [ ] 支持查看用户基本信息
- [ ] 支持修改昵称、头像
- [ ] 支持查看注册时间
- [ ] 信息修改实时生效

**业务规则**：
1. 昵称长度2-20字符
2. 头像支持JPG/PNG格式
3. 敏感信息（如微信ID）不显示

#### F004: 自选股管理

**用户故事**：
作为投资者，我想要管理我的自选股列表，以便快速查看关注股票的信息。

**验收标准**：
- [ ] 支持添加股票到自选股
- [ ] 支持从自选股删除股票
- [ ] 支持查看自选股列表
- [ ] 支持自选股排序

**业务规则**：
1. 自选股数量上限50只
2. 股票代码必须有效
3. 重复添加时提示已存在

#### F005: 板块关注管理

**用户故事**：
作为投资者，我想要关注感兴趣的板块，以便跟踪板块整体表现。

**验收标准**：
- [ ] 支持添加板块到关注列表
- [ ] 支持从关注列表删除板块
- [ ] 支持查看关注板块列表
- [ ] 支持板块分类筛选

**业务规则**：
1. 关注板块数量上限20个
2. 板块代码必须有效
3. 重复关注时提示已存在

#### F006: 偏好设置

**用户故事**：
作为投资者，我想要配置投资偏好，以便获得更个性化的投资建议。

**验收标准**：
- [ ] 支持设置风险偏好（保守/稳健/积极）
- [ ] 支持设置投资期限（短期/中期/长期）
- [ ] 支持设置关注市场（A股/港股/美股）
- [ ] 偏好设置实时生效

**业务规则**：
1. 风险偏好影响估值分析建议
2. 投资期限影响策略推荐
3. 关注市场影响数据显示范围

#### F007: 认证状态管理

**用户故事**：
作为用户，我想要保持登录状态，以便无需频繁登录。

**验收标准**：
- [ ] JWT token有效期7天
- [ ] 支持token自动刷新
- [ ] 支持主动退出登录
- [ ] 多设备登录互不影响

**业务规则**：
1. Token过期后自动跳转登录页
2. 敏感操作需要重新验证
3. 登录日志记录安全审计

## 3. 非功能需求

### 3.1 性能要求

- 登录响应时间：< 2秒
- 用户信息查询：< 500ms
- 自选股操作：< 1秒
- 并发用户数：100+

### 3.2 安全要求

- 认证授权：JWT token + 微信OAuth2.0
- 数据加密：敏感信息加密存储
- 访问控制：基于角色的权限控制
- 安全审计：登录日志记录

### 3.3 可用性要求

- 界面友好：简洁直观的用户界面
- 操作便捷：3步内完成常用操作
- 错误提示：明确友好的错误信息
- 帮助文档：关键操作提供帮助说明

### 3.4 兼容性要求

- 浏览器兼容：Chrome 90+, Firefox 88+, Safari 14+
- 设备兼容：桌面端优先，移动端适配
- 系统兼容：Windows, macOS, Linux

## 4. 数据需求

### 4.1 数据实体

| 实体名称 | 描述 | 关键属性 |
|----------|------|----------|
| User | 用户基本信息 | id, wechat_id, nickname, avatar, created_at |
| UserAuth | 用户认证信息 | user_id, token, expires_at |
| Watchlist | 自选股列表 | user_id, stock_code, stock_name, added_at |
| BoardFollow | 板块关注 | user_id, board_code, board_name, followed_at |
| UserPreference | 用户偏好 | user_id, risk_level, investment_horizon, markets |

### 4.2 数据关系

- User 1:1 UserAuth（用户与认证信息一对一）
- User 1:N Watchlist（用户与自选股一对多）
- User 1:N BoardFollow（用户与板块关注一对多）
- User 1:1 UserPreference（用户与偏好设置一对一）

### 4.3 数据流

1. 用户通过微信OAuth登录 → 创建/获取User记录 → 生成JWT token
2. 用户添加自选股 → 验证股票代码 → 写入Watchlist表
3. 用户修改偏好 → 验证参数有效性 → 更新UserPreference表

## 5. 接口需求

### 5.1 外部接口

| 接口名称 | 接口类型 | 提供方 | 调用方 | 描述 |
|----------|----------|--------|--------|------|
| 微信OAuth API | REST API | 微信开放平台 | 后端服务 | 微信登录授权 |
| 股票代码验证 | REST API | AKShare | 后端服务 | 验证股票代码有效性 |

### 5.2 内部接口

| 接口名称 | 调用方 | 被调用方 | 描述 |
|----------|--------|----------|------|
| GET /api/v1/auth/me | 前端 | 后端 | 获取当前用户信息 |
| POST /api/v1/auth/wechat/callback | 前端 | 后端 | 微信OAuth回调 |
| GET /api/v1/users/me | 前端 | 后端 | 获取用户详细信息 |
| PUT /api/v1/users/me | 前端 | 后端 | 更新用户信息 |
| GET /api/v1/users/me/watchlist | 前端 | 后端 | 获取自选股列表 |
| POST /api/v1/users/me/watchlist | 前端 | 后端 | 添加自选股 |
| DELETE /api/v1/users/me/watchlist/{code} | 前端 | 后端 | 删除自选股 |
