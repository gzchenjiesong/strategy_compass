## 1. 概述

### 1.1 模块定位

估值分析模块是Strategy Compass的分析层核心，负责计算和分析金融资产的估值水平。为投资者提供科学的估值分析工具，辅助投资决策。

### 1.2 目标用户

- 需要判断资产估值水平的投资者
- 需要估值历史对比的用户
- 需要风险评估的用户
- 需要估值区间判断的用户

### 1.3 核心价值

- 提供科学的估值分析方法
- 支持多种估值指标计算
- 提供估值历史对比分析
- 辅助投资决策

### 2.2 功能详细说明

#### F001: 估值百分位计算

**用户故事**：
作为投资者，我想要了解资产的估值百分位，以便判断估值高低。

**验收标准**：
- [ ] 支持PE历史百分位计算
- [ ] 支持PB历史百分位计算
- [ ] 支持PS历史百分位计算
- [ ] 计算结果准确可靠

**业务规则**：
1. 百分位计算基于历史数据（至少3年）
2. 过滤负值和极端值（>1000）
3. 支持不同时间窗口（1年/3年/5年/全部）
4. 计算结果保留两位小数

#### F002: 风险指标计算

**用户故事**：
作为投资者，我想要了解资产的风险水平，以便评估投资风险。

**验收标准**：
- [ ] 支持最大回撤计算
- [ ] 支持当前回撤计算
- [ ] 支持年化波动率计算
- [ ] 支持夏普比率计算

**业务规则**：
1. 最大回撤：历史最大回撤幅度
2. 当前回撤：当前价格相对历史高点的回撤
3. 年化波动率：基于日收益率计算
4. 夏普比率：（收益率 - 无风险利率）/ 波动率

#### F003: 估值区间判断

**用户故事**：
作为投资者，我想要了解资产的估值区间，以便做出投资决策。

**验收标准**：
- [ ] 支持低估区间判断（≤30%）
- [ ] 支持适中区间判断（30%-70%）
- [ ] 支持高估区间判断（≥70%）
- [ ] 判断结果准确可靠

**业务规则**：
1. 低估：百分位≤30%
2. 适中：30% < 百分位 < 70%
3. 高估：百分位≥70%
4. 支持自定义区间阈值

#### F004: 估值历史分析

**用户故事**：
作为投资者，我想要了解估值的历史变化，以便分析趋势。

**验收标准**：
- [ ] 显示估值历史趋势图
- [ ] 支持不同时间范围
- [ ] 支持估值区间标识
- [ ] 支持历史对比

**业务规则**：
1. 显示PE/PB历史百分位趋势
2. 支持1个月/3个月/1年/全部时间范围
3. 估值区间用颜色标识（低估/适中/高估）
4. 支持与当前估值对比

#### F005: 估值对比分析

**用户故事**：
作为投资者，我想要对比不同资产的估值，以便选择投资标的。

**验收标准**：
- [ ] 支持多资产估值对比
- [ ] 显示估值对比图表
- [ ] 支持估值排名
- [ ] 支持估值差异分析

**业务规则**：
1. 支持最多5个资产同时对比
2. 显示估值对比柱状图
3. 按估值百分位排名
4. 分析估值差异原因

#### F006: 估值报告生成

**用户故事**：
作为投资者，我想要获得估值分析报告，以便记录和分享。

**验收标准**：
- [ ] 自动生成估值分析报告
- [ ] 报告内容完整准确
- [ ] 支持报告导出
- [ ] 支持报告分享

**业务规则**：
1. 报告包含：估值指标、风险指标、估值区间、投资建议
2. 报告格式：Markdown或PDF
3. 支持导出为文件
4. 支持生成分享链接

#### F007: 估值提醒设置

**用户故事**：
作为投资者，我想要在估值达到阈值时收到提醒，以便及时操作。

**验收标准**：
- [ ] 支持估值阈值设置
- [ ] 支持提醒方式设置
- [ ] 支持提醒历史查看
- [ ] 提醒及时准确

**业务规则**：
1. 支持设置估值百分位阈值
2. 支持设置提醒方式（邮件、短信、推送）
3. 提醒历史保留30天
4. 提醒及时性：阈值触发后5分钟内

## 4. 数据需求

### 4.1 数据实体

| 实体名称 | 描述 | 关键属性 |
|----------|------|----------|
| ValuationData | 估值数据 | code, date, pe, pb, ps, pe_percentile, pb_percentile |
| RiskMetrics | 风险指标 | code, date, max_drawdown, current_drawdown, volatility, sharpe_ratio |
| ValuationZone | 估值区间 | code, date, zone, threshold_low, threshold_high |
| ValuationHistory | 估值历史 | code, date, pe_percentile, pb_percentile, zone |

### 4.2 数据关系

- ValuationData 1:1 RiskMetrics（估值数据与风险指标一对一）
- ValuationData 1:1 ValuationZone（估值数据与估值区间一对一）
- ValuationData 1:N ValuationHistory（估值数据与估值历史一对多）

### 4.3 数据流

1. 数据服务 → 估值计算服务 → 估值数据存储
2. 用户查询 → 估值查询服务 → 数据返回 → 前端展示
3. 估值计算 → 风险指标计算 → 估值区间判断 → 结果存储
